How This Substrate Works

# Effortless Entity Framework Execution Substrate

This is a **licensed Effortless** substrate. The C# Entity Framework data classes are produced by the `airtable-to-odxml` + `odxml-to-ef-dataclasses` transpiler pipeline (configured in `effortless.json` at `RelativePath: /licensed-effortless-tools/entity-framework-dotnet`). Lineage: the `.NET` codegen has been in use in production .NET applications since 2005 — calculated-field property getters ride on top of decades-tested CLR semantics.

## How it works

1. `inject-substrate.sh` runs `effortless -buildLocal` in `licensed-effortless-tools/entity-framework-dotnet/`. That regenerates `DataSchema.odxml` (from Airtable) and the C# DataClasses (from the ODXML).
2. `EffortlessEntityFrameworkRunner.csproj` directly globs `..\..\licensed-effortless-tools\entity-framework-dotnet\DataClasses\**\*.cs` and `SqlOnAir\DotNet\Lib\*.cs` into the build. There is no copying, no codegen here — when the licensed transpiler updates the dataclasses, this substrate is automatically in sync.
3. `Program.cs` discovers every concrete `SoAEntityBase` subclass via reflection, matches them to entities in `testing/blank-tests/*.json`, instantiates each one, copies the raw fields from JSON onto writable properties (snake_case → PascalCase), then reads every property back — including the `Formula` getters — and serializes to `test-answers/{entity}.json`.

## Why this is expected to always score 100%

The calculated fields are real C# property getters compiled from the rulebook by the Effortless transpiler. They execute on the .NET CLR, which has been the production runtime for major-bank, healthcare, and government software for two decades. There is no "evaluator" to drift; there is no formula parser to mis-implement — the formulas are just C# code.

## Caveats

- The first run will be slow because `dotnet build` downloads NuGet packages (EF Core, System.Text.Json). Subsequent runs are fast.
- Guid primary keys (auto-generated by the dataclass) are not set from the test data — the runner preserves the original string `*_id` value in the output JSON. PKs are not scored against the answer key as calculated fields.
