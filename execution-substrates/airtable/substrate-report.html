What This Substrate Does

Airtable is the oracle: the rulebook it pulls is the answer key that every other substrate is graded against. Its own test-answers are, by construction, exactly the answer keys — so it always scores 100%. A failing score here would mean the pipeline itself is broken, not the substrate.

Running [A] executes Airtable first as Step 0 so answer keys are generated from a freshly-pulled rulebook. Running [01] alone pulls without re-running the computation substrates.

Interactive Flow

  • Shows the currently active base and every known base in orchestration/bases.json.
  • Enter pulls from the active base.
  • A number switches to that base and pulls.
  • S skips the pull entirely — subsequent substrates run against the existing rulebook.
  • N adds a new base ID and pulls from it.

In CI / Docker mode it is non-interactive: keeps the current base and performs an offline sync from the cache.
