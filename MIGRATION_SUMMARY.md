## What Each File Does

| File | Purpose |
|------|---------|
| `effortless.json` | Project config: base ID, transpilers, versions, settings |
| `CLAUDE.md` | Developer instructions: how to edit, build, test this domain |
| `README.md` | Domain narrative: what the ontology models, key concepts |
| `effortless-rulebook/effortless-rulebook.json` | Core rulebook: schema, formulas, data (SSoT) |
| `execution-substrates/` | Generated by `effortless build` (do not edit) |

## How to Use the New Format

### Building a domain from rulebook:
```bash
cd rulebook-examples/customer-fullname
effortless build
```

### Pulling from Airtable (if connected):
```bash
cd rulebook-examples/jessica-basic
effortless airtabletorulebook  # Pulls from base ID in effortless.json
effortless build
```

### Editing the rulebook:
```bash
# Direct edit (no Airtable required)
vim effortless-rulebook/effortless-rulebook.json
effortless build
```

### Testing conformance:
```bash
cd rulebook-examples/star-trek
./start.sh  # or bash orchestration/orchestrate.sh
```
