# Setup: Customer FullName Rails App

The Rails app requires **Ruby 3.2.0** or later. Your system currently has Ruby 2.6.10, which is end-of-life and incompatible with modern gems.

## Quick Start

### 1. Install Ruby 3.2 using rbenv or rvm

**Using rbenv (recommended):**

```bash
# Install rbenv (if not already installed)
brew install rbenv ruby-build

# Install Ruby 3.2
rbenv install 3.2.0
rbenv local 3.2.0  # Use Ruby 3.2 in this directory

# Verify
ruby --version  # Should show 3.2.0
```

**Using rvm (alternative):**

```bash
# Install rvm
\curl -sSL https://get.rvm.io | bash -s stable

# Install Ruby 3.2
rvm install 3.2.0
rvm use 3.2.0

# Verify
ruby --version  # Should show 3.2.0
```

### 2. Install gems and start the app

```bash
cd execution-substrates/rails
bundle install
bash start.sh
```

The app will start on **http://localhost:3000**

## What the App Does

- **Models** read from PostgreSQL views (`vw_customers`)
- **Views materialize calculated fields**: `name` (LastName, FirstName) and `initials` (JS., JD., etc.)
- **Seeds** create 4 test customers from the rulebook
- **Displays** customers with all fields including calculated ones

## Troubleshooting

### "Your Ruby version is X.X.X, but your Gemfile specified 3.2.0"

You need to switch to Ruby 3.2:

```bash
# If using rbenv:
rbenv local 3.2.0
rbenv rehash

# If using rvm:
rvm use 3.2.0

# Verify
ruby --version
```

Then try again:
```bash
bundle install
bash start.sh
```

### Database connection error

Make sure PostgreSQL is running:

```bash
# Start PostgreSQL (if using Homebrew)
brew services start postgresql

# Or manually
postgres -D /usr/local/var/postgres
```

### Port already in use

If port 3000 is already in use, edit `start.sh` and change the port:

```bash
bundle exec rails server -b 0.0.0.0 -p 3001  # Use port 3001 instead
```

## Files Generated by Effortless

- `app/models/customers.rb` — Model (reads from view)
- `db/migrate/*_customers.rb` — Schema migration
- `db/views/customers.sql` — PostgreSQL view with calculated fields
- `db/seeds/customers.rb` — Test data

## How to Rebuild After Rulebook Changes

From the project root (not the rails/ directory):

```bash
# Run effortless build to regenerate all Rails artifacts
effortless build

# Then restart Rails
cd execution-substrates/rails
bash start.sh
```

All four transpilers will regenerate:
- Models (app/models/)
- Migrations (db/migrate/)
- Views (db/views/)
- Seeds (db/seeds/)
