# Rails App — Customer FullName Demo

A minimal Rails application demonstrating the Effortless philosophy: all business logic lives in the rulebook, and every substrate (Rails in this case) reads from PostgreSQL views where computed fields are materialized.

## Generated Structure

```
rails/
├── app/
│   ├── controllers/        # Rails controllers (generated)
│   │   ├── application_controller.rb
│   │   └── customers_controller.rb
│   ├── models/             # Active Record models (generated by rulebook-to-rails-models.py)
│   │   └── customers.rb    # Reads from vw_customers view
│   └── views/              # ERB templates
│       └── customers/
│           ├── index.html.erb
│           └── show.html.erb
├── config/
│   ├── application.rb      # Rails app config
│   ├── boot.rb             # Boot sequence
│   ├── routes.rb           # Routes (generated)
│   ├── database.yml        # Database config
│   └── environment.rb      # Environment setup
├── db/
│   ├── migrate/            # Migrations (generated by rulebook-to-rails-migrations.py)
│   │   └── *_customers.rb
│   ├── views/              # PostgreSQL views (generated by rulebook-to-rails-views.py)
│   │   └── customers.sql   # CREATE VIEW vw_customers
│   ├── seeds/              # Seed data (generated by rulebook-to-rails-seeds.py)
│   │   ├── customers.rb
│   │   └── seeds.rb        # Loader
│   └── schema.rb           # Schema definition
├── Gemfile                 # Ruby dependencies
├── start.sh                # Startup script
└── RAILS_APP.md            # This file
```

## Key Points

### 1. Models Read from Views

The generated `Customer` model reads from `vw_customers`:

```ruby
class Customer < ApplicationRecord
  self.table_name = 'vw_customers'
  def readonly?
    true
  end
end
```

This ensures all calculated fields (`name`, `initials`) are pre-computed by SQL and available immediately.

### 2. Calculated Fields in the Database

The rulebook defines formulas:
- **name** = `CONCAT({{LastName}}, ", ", {{FirstName}})`
- **initials** = `LEFT({{FirstName}}, 1) & LEFT({{LastName}}, 1) & "."`

These are compiled to SQL in the `vw_customers` view. Rails never recomputes them.

### 3. Single Source of Truth

All changes flow through the rulebook → rebuild cycle:

```
Edit rulebook JSON
  ↓
effortless build
  ↓
Regenerate models, migrations, views, seeds
  ↓
Rails app reflects the change
```

## Setup

### Prerequisites

- Ruby 3.2+
- PostgreSQL 12+
- Bundler

### 1. Install gems

```bash
cd rails
bundle install
```

### 2. Create .env file

```bash
cp .env.example .env
# Edit .env if your Postgres setup differs
```

### 3. Start the app

```bash
bash start.sh
```

This will:
- Create the database
- Run migrations (create table + views)
- Seed the data
- Start Rails on http://localhost:3000

## URLs

- **http://localhost:3000/** — List all customers
- **http://localhost:3000/customers/jane-smith-email-com** — View a specific customer (note the customer_id from the list)

## Demo

Edit the rulebook and rebuild to see Rails automatically reflect the change:

```bash
# In the parent directory (not rails/)
cd ../..
effortless build  # Regenerates rails/ artifacts
```

Then refresh the Rails app to see the updated schemas, models, and views.

## Notes

- The `Customers` model is read-only (bound to a view)
- Migrations are generated once on first build. To regenerate, delete `db/migrate/` and rebuild
- Seeds are generated from the rulebook data. Edit `db/seeds/` to add more data
- The view SQL is in `db/views/customers.sql`. Rails uses this when running `rails db:migrate`

## Integration with Effortless Build

The four transpilers are registered in the parent `effortless.json`:

```json
{
  "Name": "railsmodels",
  "CommandLine": "-exec ./execution-substrates/rails/gen-models.sh",
  ...
},
{
  "Name": "railsmigrations",
  "CommandLine": "-exec ./execution-substrates/rails/gen-migrations.sh",
  ...
},
{
  "Name": "railsviews",
  "CommandLine": "-exec ./execution-substrates/rails/gen-views.sh",
  ...
},
{
  "Name": "railsseeds",
  "CommandLine": "-exec ./execution-substrates/rails/gen-seeds.sh",
  ...
}
```

When you run `effortless build` from the parent directory, all four tools regenerate the Rails artifacts in order.
