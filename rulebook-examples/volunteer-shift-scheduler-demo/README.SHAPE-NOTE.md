# Shape note: this is a Naked-Claude project

This folder lives inside `rulebook-examples/` but is **not** an Effortless Rulebook (ERB) project in the canonical sense.

It is a **Naked-Claude** project — generated by Claude without a rulebook as the source of truth. It will have a different size and shape from the other examples in this directory. **That is intentional.**

## What it has

- `.xlsx` files (`rulebook.xlsx`, `volunteer-shift-scheduler-schema.xlsx`) — hand-authored schema artifacts, not rulebook JSON
- `schema/scheduler.schema.mjs` — schema as a JavaScript module
- `db/`, `server/`, `web/` — application code
- `ERB-VIBE-v1-COMPARISON.md` — comparison notes against the ERB methodology
- `VOLUNTEER-SHIFT-SCHEDULER-SPEC.md` — original spec

## What it does NOT have

- No `effortless-rulebook/<name>-rulebook.json` (no rulebook JSON as SSoT)
- No `effortless.json` (no transpiler pipeline configured)
- No conformance-tested execution substrates

## Why keep it here

It is kept as a sibling example to contrast with the rulebook-driven projects in this folder. If you are comparing "ERB approach" vs. "Naked-Claude approach" to the same domain, this is the Naked-Claude reference.

If/when this is ever ported to canonical ERB shape, the new project would live alongside this one (e.g. `volunteer-shift-scheduler/` next to `volunteer-shift-scheduler-demo/`) — not replace it.
