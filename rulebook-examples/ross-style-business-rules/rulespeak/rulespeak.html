📘 ross-style-business-rules RuleSpeak

A small, complete claims model demonstrating declarative business rules rendering themselves as RuleSpeak and tracing down to ground truth. Five rules incorporate Ronald Ross's corrections: (1) policy 'active' is ground truth, not a derived system rule; (2) high-risk uses 'must be considered ... if', not 'when'; (3) claimant of record is a priority ordering, never 'otherwise/else'; (4) valid claim is a conjunction of declarative conditions; (5) approvable is 'only if valid' (necessary, not sufficient) — not 'iff'.

ƒ Is Valid Calculated 4 hops from ground truth boolean DR-16
In RuleSpeak

A claim is considered valid if all of the following:

    • the references incident flag is set
    • any of these hold:
      • the incident claimant policy active flag is set
      • the additional claimant policy active flag is set
    • the additional claimant favorite color is “red”, or else it is not the case that the additional claimant flag is set
    • it is not the case that the claimant of record is high risk flag is set
In English

ƒ Validity Deciding Factor Calculated 5 hops from ground truth string DR-17
In RuleSpeak

The claim's validity deciding factor is determined by the following priority:

  1. “Valid — all conditions met” if the valid flag is set
  2. “No incident referenced” if it is not the case that the references incident flag is set
  3. “No active policy on incident claimant or additional claimant” if it is not the case that at least one of the following holds: the incident claimant policy active flag is set or the additional claimant policy active flag is set
  4. “Additional claimant's favorite color is not red” if all of these hold:
    • the additional claimant flag is set
    • the additional claimant favorite color is not “red”
  5. “Claimant of record is high-risk (DR-2)” if the claimant of record is high risk flag is set
  6. “Undetermined” in all other cases
In English

If is valid then "Valid — all conditions met", otherwise if not references incident then "No incident referenced", otherwise if not incident claimant policy active or additional claimant policy active then "No active policy on incident claimant or additional claimant", otherwise if has additional claimant and additional claimant favorite color lowercased is not "red" then "Additional claimant's favorite color is not red", otherwise if claimant of record is high risk then "Claimant of record is high-risk (DR-2)", otherwise "Undetermined".

As a formula
