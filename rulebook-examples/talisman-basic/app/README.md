# Special Solutions — Release Control (Talisman BASIC app)

A small full-stack app that tells the **Production Deployment Workflow** as a
business story — a software release, start to finish — where **every derived
answer is computed by the OWL/SHACL reasoner, not by application code.**

> **The one-sentence architecture:** the JSON file is the *database*; the
> `.ttl` files (OWL ontology + SHACL rules) are the *computation engine*.
> The app stores raw rows; the reasoner derives everything else.

This is the app-layer analogue of the repo's core doctrine *"the view IS the
contract — read from `vw_<entity>`, do not re-derive anything."* Here the
**reasoned RDF graph is the view**: the frontend renders what the reasoner
produced and never re-implements a formula, a closure walk, or a count.

---

## What it does

The screen walks the NTWF *Production Deployment Workflow* the way the article
frames it — and every fact on it is reasoner-derived. It stays focused on the
behaviors the article describes; it is **not** a generic ontology browser:

- **Role ↔ agent separation (Heuristic 2).** Each of the 5 steps shows its
  *role* and the *agent that currently fills it*, resolved by the reasoner
  through the role→agent assignment (RiskAnalysis-AI, James Okafor,
  Maria Gonzalez, the CI/CD pipeline, DeploymentHealth-AI). Change who fills a
  role and the screen updates with no code change.
- **The three disjoint agent types.** Humans / AI agents / pipelines, grouped —
  OWL enforces they're mutually disjoint.
- **The approval gate.** Step 3 is flagged as a first-class approval gate, not a
  plain step.
- **Transitive precedence closure.** `db.json` asserts only the 4 adjacent
  edges (1→2…4→5); the reasoner derives the full reachability (10 pairs),
  including the **never-asserted step 1 → step 5** — the article's headline
  inference.
- **Delegation / escalation chain.** Release Manager → VP Engineering is
  asserted; Release Manager → **CTO** is *inferred* by transitive reasoning.
  Role ids are resolved to display names by a lookup over the reasoned `Roles`.
- **Compliance verdict.** Whether the workflow is at compliance risk, derived
  from staleness + whether any step is AI-executed.

All of the above is served by **`GET /api/story`**, which arranges the reasoned
individuals into narrative order. It computes nothing — it reads already-derived
fields (and does name lookups over them) and lays them out.

## Architecture

```
┌────────────────────────────────────────────────────────────────────┐
│  frontend/  (Vite + React, :5173 dev)                              │
│    Renders /api/story as a release narrative (steps, agents, gate,  │
│    escalation, verdict). Computes NOTHING — only displays reasoner  │
│    output.                                                          │
└───────────────────────────────┬────────────────────────────────────┘
                                │  HTTP /api/*
┌───────────────────────────────▼────────────────────────────────────┐
│  backend/  (Express, :8088)                                        │
│    • Owns backend/db.json  ← THE DATABASE (raw, editable rows only) │
│    • For every read, shells out to the reasoner                     │
│    • Writes edit raw rows, then re-reason BEFORE persisting         │
│    • Computes NO workflow field itself                              │
└───────────────────────────────┬────────────────────────────────────┘
                                │  stdin: db.json   stdout: reasoned JSON
┌───────────────────────────────▼────────────────────────────────────┐
│  backend/reasoner/  (Python: rdflib + owlrl + pyshacl)            │
│    reason.py:                                                       │
│      1. load GENERATED ../../owl/src/ontology.owl     (TBox)        │
│      2. load GENERATED ../../owl/src/rules.shacl.ttl  (SHACL rules) │
│      3. abox_from_json.py: db.json  →  ABox triples                 │
│      4. run SHACL rules to a FIXPOINT  (calc/lookup fields)         │
│      5. run OWL-RL closure             (transitive / inverse)       │
│    → returns every individual with derived fields + competency answers│
└────────────────────────────────────────────────────────────────────┘
                                ▲
                                │  GENERATED by `effortless build`
                       owl/src/*.ttl  ←  talisman-basic-rulebook.json (SSoT)
```

### Why a JSON db works *with* the reasoner (the part that's easy to get wrong)

The committed `owl/src/individuals.ttl` (the ABox) is a **generated build
artifact** — `effortless build` rebuilds it from the rulebook every time. It is
not a writable database. So the app does **not** edit `individuals.ttl`.
Instead:

- `backend/db.json` holds **only the raw rows** (the editable data). It carries
  **no computed fields** — no `iri`, no `countAISteps`, no closure. Those aren't
  data; they're derivations.
- On each request, `abox_from_json.py` turns those raw rows into the **same ABox
  triples** the generated `individuals.ttl` would contain, and `reason.py` feeds
  them to the **generated** TBox + SHACL rules.

So the *rules* and *ontology* still come from the rulebook (via `effortless
build`); only the *individuals* come from the editable JSON. That's what makes
the app writable while keeping 100% of the computation in OWL/SHACL.

### What is raw vs. derived

| Comes from `db.json` (raw) | Derived by the reasoner (never stored) |
|---|---|
| `workflowStepId`, `displayName`, `sequencePosition`, `requiresHumanApproval`, `assignedRole`, FK refs, the 4 asserted `StepPrecedence` rows | `iri`, `relativePath`, `parentPath`, `name`, `isStale`, `monthsSinceModified`, `countAISteps`, `hasAIAgentStep`, `countOfNonProposedSteps`, the full transitive `precedesStep` / `delegatesTo` closures, OWL class disjointness |

## Run it

Prereqs: Node ≥ 18, Python 3, and `effortless build` must have generated
`../owl/src/*.ttl` at least once (the reasoner reads those).

```bash
# from app/
./start.sh            # dev:  API :8088 + Vite :5173  → open http://localhost:5173
./start.sh prod       # prod: builds the UI, Express serves API+UI on :8088
```

`start.sh` preflights the generated `.ttl` and the Python deps, frees its ports,
then boots. (Per repo doctrine, restart is one command — it kills its ports
first.)

### Run pieces individually

```bash
# backend only
cd backend && npm install && PORT=8088 npm start

# frontend only (proxies /api → :8088)
cd frontend && npm install && npm run dev

# the reasoner, standalone (the computation engine, no web layer)
python3 backend/reasoner/reason.py backend/db.json | less
```

## API

| Method | Path | What |
|---|---|---|
| GET | `/api/health` | builds the graph; returns reasoned triple count |
| GET | `/api/story` | the release narrative the UI renders (steps→agents, gate, escalation, team, closure, verdict) — all reasoner-derived |
| GET | `/api/db` | the **raw** db.json (editable rows, no derivations) |
| GET | `/api/reasoned` | everything: individuals + derived fields + competency answers |
| GET | `/api/individuals/:cls` | reasoned individuals of one class (e.g. `Workflows`) |
| GET | `/api/competency` | closures, delegation, disjoint classes, role→agent |
| POST | `/api/individuals/:cls` | add a **raw** row (re-reasons before persisting) |
| PATCH | `/api/individuals/:cls/:id` | patch a row's raw fields by primary key |

Writes that include a derived field (e.g. `countAISteps`) are **rejected with
400** — the db stays raw-only.

## Doctrine this app honors

- **No silent fallbacks.** If the `.ttl` is missing, a reference dangles, or the
  SHACL rules don't reach a fixpoint, the reasoner **raises** and the API
  returns the error verbatim (500). It never serves raw, unreasoned numbers in
  place of a real answer.
- **The reasoned graph is the contract.** The frontend reads derived fields; it
  never re-derives them. No JS formula parser, no closure walk in app code.
- **No bespoke cache.** The backend re-reads `db.json` and re-reasons from
  scratch on every request — the db content changes between requests, so there
  is nothing to cache. The fixpoint loop iterates on the graph's *content*
  (a triple-set fingerprint), not its triple count, so a value that flips
  `true→false` once its input resolves — like `isAtComplianceRisk` — converges
  correctly instead of stopping one pass early.
- **db.json is mock data, reset-friendly.** Like the rest of this project, the
  seed rows are demo data; curating them is normal. The *rules* (the `.ttl`,
  generated from the rulebook) are the sacred part — and this app never edits
  them.

## A transpiler bug this app surfaced (and fixed at the source)

Building this app's interactive compliance verdict exposed a real bug in the
`rulebook-to-owl` transpiler: every cross-entity lookup was emitted as
`BIND(STR(?x) AS ?_result)`, stringifying the copied value. So
`ComplianceVerdicts.isStale` became the *string* `"false"`, and since a
non-empty string is truthy in SPARQL, `isAtComplianceRisk = isStale AND
hasAIExecutedStep` was **always true** — the verdict could never read "OK".

The fix was made where the bug lived — the substrate generator
(`execution-substrates/owl/inject-into-owl.py`), not in app code: a standalone
lookup now returns the bare value (datatype preserved), and `STR()` is applied
only at the `CONCAT` string-building boundary. `effortless build` regenerates
`owl/src/rules.shacl.ttl` with `BIND(?j1 AS ?_result)`; the data files are
byte-for-byte unchanged. The OWL↔Postgres conformance suite still passes 13/13.

Two smaller, faithful notes:

- **Date typing.** The reasoner's date math (`monthsSinceModified`, `isStale`)
  needs typed temporal literals. The JSON→ABox serializer emits ISO date(Time)
  strings as `xsd:dateTime`/`xsd:date` so `YEAR()`/`NOW()` resolve — faithful
  typing of a value that genuinely is a dateTime, not a guess.
- **Multi-criteria aggregations.** A few chained `COUNTIFS` rollups
  (`countEngineeringOwnedSteps`, `countLegalOwnedSteps`) still evaluate to `0` —
  a separate, known `rulebook-to-owl` limitation. The app surfaces whatever the
  `.ttl` computes; when the transpiler's multi-criteria handling is fixed, those
  populate with no app change.
